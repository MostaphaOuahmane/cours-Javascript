JavaScript -II/01 introduction

Constantes & variables

1- Qu'est qu'une variable

Une variable est un conteneur servant à stocker des informations de manière temporaire, du texte (une chaîne de caractère) ou un nombre. La variable varie ! Elle stocke différentes valeurs au fil du temps.

Une variable et la valeur sont deux choses différentes

2- Régles de déclaration

On dit (comme en php) que l'on "déclare" une variable. On déclare une variable avec les mots-clefs :

  • let
  • var
  • Le nom d'une variable doit obligatoirement commencer par une lettre ou un underscore _
  • Le nom d'une variable ne doit contenir que des lettres, des chiffres et des underscore
  • il n'y a pas d'espace
  • cf. introduction.js et Entrainement

3- initialiser une variable-

Quand on assigne une valeur à une variable vide on dit que l'on stocke une valeur ou qu'on initialise une variable

4 - Différance entre varet let

let el plus récent et plus contraignant que var

A - la remontée ou hoisting des variables

avec var on n'est pas obligé de déclare la variable avant de la manipuler ou l'affecter

B - la redéclaration des variable

avec var on avait le droit de déclarer plusieurs fois une même variable en utlisant à chaque fois var (ce qui avait pour effet de modifier la valeur stockée ). On peut pas me faire avec let

C - la "portée" d'une variable

la "portée" d'une variable désigne l'endroit où cette variable va pouvoir être utlisée dans un script.

Nous reparlerons donc de cette portée des variable lorsque nous aborderons les fonctions en JavaScript

D- Conclusion : let ou var ?

La syntaxe de déclaration des variables avec let correspond à la nouvelle syntaxe. La syntaxe avec var est l’ancienne syntaxe qui est vouée à disparaitre.

Vous devriez donc aujourd’hui toujours utiliser le mot clef let pour déclarer vos variables, sachant que c'est un peu contraignant.

5- Quelle utilité pour les variables ?

Il sera pratique de stocker différents types de données ou d'informations dans une variable.
